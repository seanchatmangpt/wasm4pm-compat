# Towards Nanosecond-Scale Autonomic Process Mining: Architectural Limits and Methodological Advancements in the `wasm4pm` Ecosystem

**A Formal Dissertation on the Optimization of Edge-Native Process Intelligence**

## Abstract
The `wasm4pm` ecosystem successfully introduces deterministic, nanosecond-scale process mining within a WebAssembly-native execution runtime, strictly bounded by the `wasm4pm-compat` ontological type laws. It enforces structural integrity, monotonic evidence admission, and cryptographic provenance. However, the current implementation relies on algorithmic compromises necessary for its initial baseline. This dissertation identifies four critical areas for advancement: (1) The State-Space Explosion in Uninformed A* Alignments, (2) The Hash-Collision Bottleneck in SIMD Vectorization, (3) The Dimensionality Limits of Tabular Q-Learning in Autonomic Orchestration, and (4) The Absence of Behavioral Soundness Verification at the Admission Boundary. Resolving these limitations is necessary for the next generation of real-time, highly concurrent autonomic process mining.

## Chapter 1: The Alignment Bottleneck and Informed Heuristics
**Context:** `wasm4pm/wasm4pm/src/alignments.rs`

Optimal conformance checking is currently achieved via an A* search algorithm over the product space of the trace and the Petri net marking. While the algorithm correctly bounds iterations to `100,000` to prevent infinite livelocks, it utilizes an uninformed, albeit admissible, heuristic function ($h(x) = 0.0$). 

**The Limitation:**
Using $h(x) = 0.0$ reduces the A* algorithm to Dijkstra’s algorithm. In highly parallel, concurrent models (such as those generated by the Inductive Miner), the state space expands exponentially. The search explores thousands of symmetrical interleavings with equal cost before progressing toward the final marking.

**Proposed Advancement:**
The implementation must migrate to an **Informed Structural Heuristic**. By pre-computing the shortest path to the final marking using the Marking Equation ($M = M_0 + W \cdot \vec{\sigma}$) or utilizing a distance matrix on the structural Directly-Follows Graph, $h(x)$ can tightly estimate the remaining cost. If $h(x)$ strictly underestimates the true cost, A* maintains formal optimality while pruning the exponential branching factor of concurrent invisible transitions.

## Chapter 2: The Scalar Barrier in SIMD Streaming
**Context:** `wasm4pm/wasm4pm/src/simd_streaming_dfg.rs`

The current streaming architecture masterfully utilizes WebAssembly `v128` SIMD intrinsics to accumulate node frequencies (`i32x4_add`), achieving ~50ns overhead per event.

**The Limitation:**
While node frequency accumulation is fully vectorized, edge counting remains bound to scalar execution. The `add_trace()` hot path relies on an `FxHashMap<(u32, u32), usize>`. Because hash map lookups involve non-deterministic collision resolution and unpredictable memory access patterns, they cannot be vectorized into a single SIMD instruction. The current mitigation—loop unrolling—merely hides branch prediction penalties but does not achieve true Data-Level Parallelism (DLP).

**Proposed Advancement:**
Transition from hash maps to **Flattened Adjacency Matrices** or **Perfect Hashing** for edge accumulation. Since the vocabulary size $|A|$ is often relatively small ($|A| < 1000$), pre-allocating a flat 1D array of size $|A|^2$ allows the engine to compute the index deterministically: `index = from_id * |A| + to_id`. This eliminates the hashing overhead entirely and permits the use of `v128` gather/scatter intrinsics to increment multiple edge frequencies simultaneously.

## Chapter 3: Dimensionality Limits of Tabular Q-Learning
**Context:** `wasm4pm/wasm4pm/src/autoprocess.rs` & `rl_orchestrator.rs`

The MAPE-K autonomic loop utilizes a branchless decision engine driven by a pre-computed Q-Table. The state space is bounded to 8 dimensions, resulting in exactly 368,640 discrete states and requiring ~9.2 MB of persistent memory per agent.

**The Limitation:**
Tabular Q-Learning suffers fundamentally from the "Curse of Dimensionality." The current 8D state vector requires aggressive quantization (e.g., binning event rates into 8 discrete buckets). This quantization destroys granular continuous data, preventing the Reinforcement Learning agent from perceiving subtle, high-frequency systemic drifts. Furthermore, the 9.2 MB memory footprint per agent limits the scalability of the system when deploying hundreds of parallel agents in constrained IoT environments.

**Proposed Advancement:**
The orchestration layer must evolve from Tabular Q-Learning to **Linear Function Approximation** or **Lightweight WASM Neural Networks (Deep Q-Learning)**. By representing the policy as a set of weights ($\vec{w}$) rather than a discrete matrix, the agent can consume continuous state vectors natively. This reduces the memory footprint from $O(|S| \times |A|)$ to $O(|Features| \times |A|)$ (e.g., from 9.2 MB to mere kilobytes), while dramatically improving the agent's ability to generalize learning across previously unseen states.

## Chapter 4: Behavioral Soundness Verification
**Context:** `wasm4pm-compat/src/petri.rs` & `wasm4pm-compat/src/models.rs`

The `wasm4pm-compat` boundary currently implements `is_structural_workflow_net()`. This successfully verifies that the model has a single source, a single sink, and no unconnected nodes.

**The Limitation:**
Structural correctness does not guarantee behavioral liveness. A model may be a valid structural WF-net but still contain dead transitions (activities that can never fire), livelocks (infinite $\tau$-loops), or improper completion (reaching the sink place with residual tokens left in parallel branches). Currently, these behavioral flaws are only caught at runtime (e.g., when the token replay iteration budget is exhausted).

**Proposed Advancement:**
The admission boundary must be upgraded to enforce **Formal Behavioral Soundness**. This requires the implementation of a Coverability Graph (or Reachability Tree) generator at the `wasm4pm-compat` boundary. Evidence admission should computationally prove the three Van der Aalst soundness properties:
1. **Proper Completion:** The final marking is the *only* marking containing a token in the sink place.
2. **Option to Complete:** From any reachable marking, it is always possible to reach the final marking.
3. **No Dead Transitions:** Every transition in the network can be fired in at least one firing sequence.
Models failing these proofs must be formally refused by the `Admit` trait before reaching the execution engine.

## Conclusion
The `wasm4pm` ecosystem represents the vanguard of deterministic, edge-native process mining. By formalizing structural bounds and leveraging memory-safe compilation, it has established a secure foundation. However, to achieve true theoretical optimality, the system must address the A* heuristic vacuum, break the scalar hashing bottleneck in streaming matrices, replace discrete Q-tables with continuous function approximation, and elevate its structural verifications to encompass full behavioral soundness. Implementing these methodologies will solidify `wasm4pm` not merely as a compatibility layer, but as the definitive autonomic process intelligence runtime.
