# Retired Features Historical Drift Report — v26.6.5

This report documents the scan of the repository for references to retired features (`ts` and `wasm`) of the compatibility crate (`wasm4pm-compat`). As part of the transition to a strict three-feature limit (`formats`, `strict`, `wasm4pm`), the features `ts` and `wasm` and their associated dependencies (`specta`, `tsify`, `wasm-bindgen`, and `serde-wasm-bindgen`) have been completely retired from compilation.

This report serves as a formal tombstone catalog for downstream auditing and code-generation tools to note the retired state of these capabilities.

---

## 1. Executive Summary & Retirement Notice

The features `ts` and `wasm` are **permanently retired**. 
* **`ts`**: Gated TypeScript interface generation. Formerly used `specta` and `tsify` to generate type definitions (`.d.ts`) directly from Rust evidence structures.
* **`wasm`**: Gated WebAssembly boundary (marshaling) DTOs, functions, and memory structures for JS/TS interop via `wasm-bindgen`.

To enforce compilation speed, reduce dependencies, and establish clean boundaries, all Rust modules and tests related to these features were removed from the `src/` and `tests/` directories. Downstream tools, CI pipelines, and generation scripts must ignore these features.

## 3. Catalog of Historical Drift in Code Generation & Auditing (`ggen/`)

The `ggen/` directory contains template configurations, scripts, and rules that historically managed the generation of these boundary packages. The following files are marked as **Obsolete/Retired Substrate**:

### 3.1. Projection Manifests
* **`ggen/projections/ts.projection.yaml`**: Legacy manifest defining how to project Rust type law generic shapes into TypeScript.
  * *Gate*: Gated by feature `ts`.
  * *Note*: This file is kept purely for reference to design the typescript mapping downstream; it is ignored by active `ggen` rules.
* **`ggen/projections/wasm.projection.yaml`**: Legacy manifest mapping out the WASM boundary DTOs.
  * *Gate*: Gated by feature `wasm`.
  * *Note*: Ignored by active `ggen` rules.

### 3.2. Projection Rules & Specifications
* **`ggen/rules/ts-projection-law.yaml`**: Declares rules and test fixtures (like `LAW 1: No PhantomData at the boundary`) for TypeScript translation.
* **`ggen/rules/wasm-boundary-law.yaml`**: Declares WebAssembly boundary rules and restrictions.
* **`ggen/audits/audit-ts-projection.sh`**: Legacy shell script checking generated TypeScript boundaries.
* **ggen/intel/tsify-capability-map.md**: Lists tsify configurations.
* **ggen/intel/wasm-abi-map.yaml**: Maps out WASM ABI types.
* **ggen/intel/wasm-boundary-prohibited.yaml**: Contains rules against generic structures at WebAssembly boundaries.

### 3.3. Tera Templates
* **`ggen/templates/ts-projection.rs.tera`**: Used to render Rust code representing tsify models.
* **`ggen/templates/wasm-projection.rs.tera`**: Used to render WASM endpoint wrappers.
* **`ggen/templates/audit-ts-projection.sh.tera`**: Tera template for the TS audit script.

## 4. Tombstone & Downstream Tool Guidelines

Downstream tools must behave as follows:

1. **Feature Check-CFG warnings (`unexpected_cfgs`)**: Downstream crates compile under `unexpected_cfgs` which will warn if they pass `--cfg feature="ts"` or `--cfg feature="wasm"`.
2. **Ignored Targets**: `ggen sync` runs without executing any rule for `ts-projection` or `wasm-projection`.
3. **No Optional Dependencies**: Cargo builds must never assume or specify `tsify`, `specta`, or `wasm-bindgen` features for `wasm4pm-compat`.
