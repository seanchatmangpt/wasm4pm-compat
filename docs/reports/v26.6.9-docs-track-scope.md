## 2. Layout Enforcement & Write-Path Access Control

To maintain strict segregation of duties between documentation subagents and code/logic modifications, the following write-path restrictions are enforced:

### Allowed Write Paths (Documentation & Research Track)
* `docs/**/*` — Core documentation files (Guides, Explanations, Reference, Reports).
* `research/**/*` — Deep-dive process theory research.
* `.agents/scope_drift_worker/**/*` — Workspace files, briefs, and heartbeats for this agent.

### Forbidden Write Paths (Documentation & Research Track)
* `src/**/*` — Production source code.
* `Cargo.toml` — Crate configuration and dependencies.
* `tests/**/*` — Unit, integration, and trybuild UI tests.
* `examples/**/*` — Runnable example programs.
* `ggen/**/*` — Code generation substrate, schema definitions, template files, and queries.
* `c8-*/**/*` — Subcrates (`c8-adversary`, `c8-instruments`, `c8-market`, `c8-receipts`, `c8-time`).
* `emitted/**/*` — Static audit records and results (which belong to the verification process).
* `target/**/*` — Build output directory.

## 3. Code, Test, and Source Issues ("CLAUDE_TRACK")

The following issues were identified during scanning and auditing. Because they require editing files in forbidden directories (such as `src/`, `Cargo.toml`, or `tests/`), they **cannot** be resolved by this documentation track and are labeled as `CLAUDE_TRACK`. They must be routed to the logical code/test implementation track:

### CLAUDE_TRACK_001: Manifest Version Discrepancies in Subcrates and Code
* **Observation**: While the root package `wasm4pm-compat` Cargo.toml has version `26.6.9`, subcrates like `c8-receipts` and files like `c8-receipts/src/implementation.rs` (lines 89, 97, 105, 124, 138) hardcode or expect `"26.6.9"`.
* **Required Code Action**: Synchronize logical version constants in subcrate source code files to `"26.6.9"` once the freeze constraints are resolved.
* **Impact Path**: `c8-receipts/src/implementation.rs`, `c8-time/Cargo.toml`, `c8-receipts/Cargo.toml`, `c8-market/Cargo.toml`, `c8-instruments/Cargo.toml`, `c8-adversary/Cargo.toml`.

### CLAUDE_TRACK_002: Dead Submodules and Feature Check Warnings (`unexpected_cfgs`)
* **Observation**: The features `ts` and `wasm` have been retired from `Cargo.toml`. While the directories `src/ts/` and `src/wasm/` and their respective tests in `tests/graduation.rs` have been removed in recent cleanups, checking for configuration drift during script runs or workspace auditing is still needed. (Note: Compilation warnings may still occur if any scripts or tool configurations expect these features).
* **Required Code Action**: Verify that the compiler's unexpected config checker (`unexpected_cfgs`) is configured in Cargo.toml or clippy.toml to avoid diagnostic noise on migrated workspace targets.
* **Impact Path**: `Cargo.toml`, `clippy.toml`.

### CLAUDE_TRACK_003: Stale Feature Verification in `ggen` Audit Scripts
* **Observation**: The script `ggen/audits/audit-feature-isolation.sh` contains hardcoded checks verifying that `ts` and `wasm` modules are gated by `#[cfg(feature = "ts/wasm")]` in `src/lib.rs`, and that `serde`, `specta`, `tsify`, and `wasm-bindgen` are listed as optional dependencies in `Cargo.toml`. Since these features and dependencies are deleted, this shell script fails validation (exit code 1).
* **Required Code Action**: Refactor the verification logic in `ggen/audits/audit-feature-isolation.sh` and the corresponding specification `ggen/audits/AUDIT_SPEC.md` to remove the assertion that retired features/modules exist.
* **Impact Path**: `ggen/audits/audit-feature-isolation.sh`, `ggen/audits/AUDIT_SPEC.md`.

### CLAUDE_TRACK_004: Generated-Caste Banners in Source Code
* **Observation**: `src/witnesses.rs` contains comment banners stating it was "Generated by ggen sync", which violates the project's peer-source doctrine.
* **Required Code Action**: Update the `ggen` template `witness-marker.tera` to output a peer-source-compatible header that treats the file as a first-class source file, then re-run `cargo make ggen-witnesses`.
* **Impact Path**: `src/witnesses.rs`, `ggen/templates/witness-marker.tera`.
