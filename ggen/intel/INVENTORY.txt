KEY ECOSYSTEM INVARIANTS
================================================================================

1. Feature Model
   • Exactly 3 public user features: formats, strict, wasm4pm
   • No per-format flags (all data models always-on)
   • Additional features (ts, wasm) are internal projection gates

2. Nightly-First
   • No stable Rust support; applications conform upward to type law
   • Required features: generic_const_exprs, adt_const_params, const_trait_impl,
                        min_specialization, portable_simd

3. Type-Law-First
   • Every ecosystem component (OCEL, XES, BPMN, Petri, POWL, etc.) has a witness marker
   • Witness markers are zero-sized PhantomData tags
   • State tokens (Raw, Parsed, Admitted, etc.) are lifecycle enforcement

4. Zero Runtime Dependencies
   • All ecosystem components are compile-time or opt-in feature-gated
   • TypeScript/WASM machinery removed at compile time for non-ts/wasm builds

5. Structure-Only Scope
   • No discovery, conformance checking, token replay, alignment
   • All execution engines graduate to wasm4pm
   • This crate is admission + projection layer only
