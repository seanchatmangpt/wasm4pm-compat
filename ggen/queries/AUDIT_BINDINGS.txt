## FORBIDDEN PATTERNS (5 total)

### Pattern 1: condition-cell-overflow
  Description: Declaring or using ConditionCell<BITS> with BITS > 8, violating the 'Need 9 means split' law. The Blue River Dam covenant limits condition cells to 8 bits.
  Forbidden By: https://wasm4pm-compat.rs/ontology#ConditionCellOverflowLaw
  Allowed In: (none)
  Scope: Crate-wide; compile-time law enforced via generic_const_exprs

### Pattern 2: raw-evidence-exported-as-admitted
  Description: Exporting or returning Evidence<T, Raw, W> as if it were Admitted, or using a Raw-state evidence where Admitted is required. This violates the one-way-door law.
  Forbidden By: https://wasm4pm-compat.rs/ontology#RawExportedAsAdmittedLaw
  Allowed In: (none)
  Scope: Crate-wide; compile-fail law (E0308 type mismatch)

### Pattern 3: silent-structure-loss
  Description: Performing a lossy transformation (e.g., OCEL → XES, removing object links) without emitting a LossReport. The loss is hidden from audit.
  Forbidden By: https://wasm4pm-compat.rs/ontology#CompatDiagnostic
  Allowed In: named-projection
  Scope: Module-level exception (src/loss.rs); runtime enforcement via LossReport carrier

### Pattern 4: soundness-forged
  Description: Constructing or manipulating a WfNetConst<SoundnessWitnessed> without passing through wasm4pm's soundness verification. Soundness witnesses are non-forgeable.
  Forbidden By: https://wasm4pm-compat.rs/ontology#SoundnessForgedLaw
  Allowed In: (none)
  Scope: Crate-wide; type-level seal (Witness path is crate-private)

### Pattern 5: witness-discrimination-violation
  Description: Using Admission<T, W1> in place of Admission<T, W2>, or merging evidence admitted against different witnesses without proper re-admission. Witness tags are sealed.
  Forbidden By: https://wasm4pm-compat.rs/ontology#WitnessDiscriminationLaw
  Allowed In: (none)
  Scope: Crate-wide; compile-fail law (E0308 type mismatch)

## QUERY EXECUTION NOTES

Both queries return bindings in long-form (one row per relationship) rather than aggregated:
- select-forbidden-patterns.rq: One row per (pattern, allowed_context) pair; patterns without contexts return single row with NULL ?contextName
- select-allowed-contexts.rq: One row per (context, forbidden_pattern) pair; contexts without links return single row with NULL ?forbiddenPatternName

This design allows audit template to iterate through all bindings without aggregation complexity.
