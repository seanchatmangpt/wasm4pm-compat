### Manifest Note

A root-level `ggen-witness.toml` was created to resolve the ggen security policy
that blocks `../` traversal when the manifest lives in `ggen/`. The root manifest
mirrors the same ontology sources and witness-markers rule with output at
`src/witnesses.rs` (no traversal needed from root).

The `[inference]` section in `ggen/ggen.toml` was updated to use an inline CONSTRUCT
query (`"CONSTRUCT { ?s ?p ?o } WHERE { ?s ?p ?o } LIMIT 1"`) because ggen v26.5.21
treats the `construct` field as inline SPARQL, not a file path — resolving the
"error at 1:10: expected CONSTRUCT" parse failure on every prior attempt.

## Bind Site

```rust
/// Witness markers and witness families (type-level proof carriers).
#[macro_use]
pub mod witness;
/// Compiled witness marker declarations — one entry per WitnessMarker in the ontology.
/// Generated by `ggen sync --rule witness-markers`; maintained via TTL + ggen, not by hand.
pub mod witnesses;
```

The `#[macro_use]` attribute on `mod witness` brings `witness_marker!` into scope for
`witnesses.rs` without requiring `#[macro_export]` on the hand-written macro.
